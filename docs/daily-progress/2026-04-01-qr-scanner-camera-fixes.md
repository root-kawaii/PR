# 2026-04-01: QR Scanner — iOS Camera Fixes & Mobile Modal

**Branch**: `17-gestionale-qr-code`
**Status**: In Progress

---

## Overview

Fixed multiple issues with the QR scanner camera on the dashboard:

1. **iOS Safari camera not starting** — two root causes identified and fixed:
   - Dynamic `import('html5-qrcode')` inside the click handler broke the user-gesture chain required by iOS Safari to allow `getUserMedia`.
   - iOS Safari blocks `getUserMedia` on non-HTTPS origins; the dev server was reachable only via `http://192.168.x.x`.
2. **Camera overlay blocking UI** — the `html5-qrcode` video container was not properly contained, causing it to overflow the card and intercept pointer events on elements below.
3. **Mobile UX** — on small screens the inline camera viewer is too cramped; replaced with a fullscreen modal.

---

## Changes

### Dashboard (pierre_dashboard)

#### `vite.config.ts`

Added `@vitejs/plugin-basic-ssl` and `server.host: true` so the dev server is reachable from physical devices over HTTPS on the local network:

```ts
import basicSsl from '@vitejs/plugin-basic-ssl'

export default defineConfig({
  plugins: [react(), tailwindcss(), basicSsl()],
  server: { https: true, host: true },
})
```

#### `src/pages/QRScannerPage.tsx`

**iOS dynamic-import fix**: replaced lazy `await import('html5-qrcode')` with a top-level static import. Dynamic import inside an async click handler breaks the Safari user-gesture association for camera access.

**Desktop containment fix**: the `#qr-reader-inline` div now uses `position: relative` + `overflow: hidden` + explicit `minHeight: 300px` when active. Previously the div was hidden with `display: none` (Tailwind `hidden` class), which also caused `html5-qrcode` to mount into a zero-size container.

**Dynamic element ID**: `Html5Qrcode` is instantiated on `'qr-reader-inline'` (desktop) or `'qr-reader-modal'` (mobile) depending on a `window.matchMedia('(max-width: 767px)')` check at click time. This avoids DOM manipulation to move a single element between the inline position and the modal.

**`requestAnimationFrame` barrier**: after setting state that shows the target div, a single `requestAnimationFrame` ensures the DOM has updated before `Html5Qrcode` mounts into it.

**Mobile fullscreen modal**: when `isMobile` is true, a `fixed inset-0 bg-black z-50` overlay opens with the camera filling the screen. Closing the modal (X button or successful scan) stops the scanner and dismisses the overlay.

**Error surfacing**: the `catch` block now exposes `err.message` instead of a fixed generic string, making it easier to diagnose future camera issues.
